# 1.1.2. 探索的データ解析の古典的データ解析との相違

#### データ解析の手法

EDAはデータ解析の手法の一つである。他にどのようなデータ解析手法があり、EDAはそれらとどのように異なるか。代表的なデータ解析手法は次の3つである：

1. 古典的
2. 探索的（EDA）
3. ベイズ


#### 解析手法の理論的枠組み

これら3つの手法は、いずれも一般的な科学／工学の問題から始まり、科学／工学の結論を導く点で似ている。違いは中間段階の順序と焦点にある。 

古典的解析では、順序は次の通りである

	問題 => データ => モデル => 解析 => 結論

EDAの場合、順序は

	問題 => データ => 解析 => モデル => 結論

ベイズ法の順序は次の通りである

	問題 => データ => モデル => 事前分布 => 解析 => 結論 

#### データの基盤モデルを扱う方法による3手法の区別

古典的な解析では、データ収集の後にモデル（正規性、線形性など）が仮定され、続く解析、推定、検定はそのモデルの母数に焦点を当てる。一方、EDAでは、データ収集の後にモデルを仮定するのではなく、適切なモデルを推測することを目的とした解析が直ちに続く。最後に、ベイズ解析では、解析者は科学的／技術的知識・専門性を解析に組み込むため、選択したモデルの母数に対してデータに依存しない分布を仮定する。したがって、この解析は、母数に対する事前分布と収集されたデータを形式的に組み合わせ、モデルの係数に関する推論や仮定の検定を組みあわせて行うことで構成される。 

現実世界では、データ分析者は上記3つの手法（およびその他の手法）の要素を自由に組み合わせている。上記の区別は、3つの手法間の主要な差異を強調するために設けられたものである。
